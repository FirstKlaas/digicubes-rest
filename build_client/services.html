All role services

add_right(role: digicubes.client.proxy.role_proxy.RoleProxy, right: digicubes.client.proxy.right_proxy.RightProxy) → bool¶

Adds a right to this role.

Because rights and roles have a many-to-many relation, this equivalent to adding a role to the right. Therefor the corresponding method from the right service is called.

all() → Optional[List[digicubes.client.proxy.role_proxy.RoleProxy]]¶

Returns all roles

The result is a list of RoleProxy objects

Creates a new role

create_bulk(roles: List[digicubes.client.proxy.role_proxy.RoleProxy]) → None¶

Create multiple roles

delete_all()¶

Removes all roles from the database.This operation is atomic. A successful operation is indicated by a 200 status. If the operation fails, a ServerError is thrown.

Warning

This operation cannot be undone. So be shure you know, what you are doing.

get(role_id: int) → Optional[digicubes.client.proxy.role_proxy.RoleProxy]¶

Get a single user.

The requested user is specified by the id. If the requested user was found, a UserProxy object will be returned. None otherwise.

remove_right(role: digicubes.client.proxy.role_proxy.RoleProxy, right: digicubes.client.proxy.right_proxy.RightProxy) → bool¶

Removes a right from this role.

Because rights and roles have a many-to-many relation, this equivalent to removing a role from the right. Therefor the corresponding method from the right service is called.

RightService¶

class digicubes.client.service.RightService(client: Any)¶

The rights service

add_role(right: digicubes.client.proxy.right_proxy.RightProxy, role: digicubes.client.proxy.role_proxy.RoleProxy) → bool¶

Add a role to this right. The role and the right must exist. If not, a DoesNotExist error is raised.

all() → List[digicubes.client.proxy.right_proxy.RightProxy]¶

Returns all rigths. The result is a list of RightProxy objects.

clear_roles(right: digicubes.client.proxy.right_proxy.RightProxy) → bool¶

Clears all roles from the right. After a succesful call no role is associated with this right. The right must exist. At least the id of the right has to be set.

Parameters

right (RightProxy) – The right, where the roles should be cleared.

Return bool

True, if the operation was successful, False else.

delete_all()¶

Delete all digicube rights. This operation is atomic. A successful operation is indicated by a 200 status. If the operation fails, a ServerError is thrown.

Warning

This operation cannot be undone. So be shure you know, what you are doing.

get(right_id: int) → Optional[digicubes.client.proxy.right_proxy.RightProxy]¶

Get a single right by id

remove_role(right: digicubes.client.proxy.right_proxy.RightProxy, role: digicubes.client.proxy.role_proxy.RoleProxy) → bool¶

Removes a role from this right. Both, the role and the right must exist. If not, a DoesNotExist exception is thrown.
